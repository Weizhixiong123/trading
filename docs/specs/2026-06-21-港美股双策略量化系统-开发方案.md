# 港美股双策略量化系统 — 开发方案文档

> 版本：v1.0  ·  日期：2026-06-21
> 用途：内部技术选型/排期 + 对客户交付说明
> 范围：策略一【日内 MACD 全自动交易】、策略二【大级别中长线选股持仓】

---

## 1. 项目概述

为客户开发一套**港股 / 美股通用**的量化交易系统，包含两套**完全独立、可单独开关**的策略：

| | 策略一 | 策略二 |
|---|--------|--------|
| 定位 | 日内 MACD 全自动交易 | 大级别中长线选股持仓 |
| 周期 | 15min 主 + 5min 确认 | 月线/周线选股 + 日线择时 |
| 交易频率 | 高频日内、当日清仓不隔夜 | 低频、持仓 1~6 个月、月度调仓 |
| 自动化 | 全自动开/平/止盈止损 | 全自动选股+提醒、半自动建仓、全自动调仓 |
| 市场 | 港股、美股全标的 | 港股、美股全标的 |

最终交付一套：**回测 → 模拟盘 → 实盘**链路完整、带前端面板、带交易日志、参数可后台调整的系统。

---

## 2. 技术选型（已确定）

**框架：vnpy（VeighNa）  ·  券商通道：富途 + 老虎（双通道）**

选型理由：在所有开源量化框架中，**只有 vnpy 同时拥有富途和老虎的成熟官方网关**，正好命中"双通道都要"的硬需求。其它框架（backtrader / RQAlpha / QUANTAXIS / zipline）都需自行开发富途/老虎对接，等于多花数周且不稳定。

需求 → vnpy 模块映射：

| 需求 | 对应模块 | 现成度 |
|------|----------|--------|
| 富途下单/行情 | `vnpy_futu`（底层 futu-api + 本地 FutuOpenD） | 现成 |
| 老虎下单/行情 | `vnpy_tiger`（底层 tigeropen） | 现成 |
| 策略一 日内 MACD | `vnpy_ctastrategy`（CTA 引擎，自带止盈止损/定时器） | 框架现成，信号逻辑自写 |
| 策略二 中长线持仓 | `vnpy_portfoliostrategy`（组合策略） | 框架现成 |
| 回测 + 参数优化 | `vnpy_ctabacktester`、组合回测引擎 | 现成 |
| 数据存储/管理 | `vnpy_sqlite`/`vnpy_mysql` + `vnpy_datamanager` | 现成 |
| 交易日志 | 引擎自带 + `vnpy_datarecorder` | 现成 |
| 参数后台修改 | CTA 引擎参数 JSON 化、可热改 | 现成 |
| 前端面板 | `vnpy_webtrader`（够用但卖相一般）/ 自研 | 部分现成 |

**需自行开发（任何框架都需要）**：①MACD 背离的数学化识别；②策略二的全市场选股扫描脚本；③客户级前端。

## 3. 系统架构

分层设计，各层职责单一、接口清晰：

```
┌─────────────────────────────────────────────┐
│  前端面板 (Vue 后台模板 + TradingView 图表)      │  ← 账户/持仓/信号/日志展示、参数配置、策略开关
├─────────────────────────────────────────────┤
│  Web 接口层 (FastAPI: REST + WebSocket)         │  ← 暴露查询/控制接口给前端
├─────────────────────────────────────────────┤
│  策略引擎层 (vnpy)                                │
│   ├─ 策略一: 日内 MACD (CtaTemplate)             │
│   ├─ 策略二: 中长线持仓 (PortfolioTemplate)       │
│   ├─ 选股扫描模块 (独立脚本/服务)                  │
│   ├─ 风控模块 (单日止损/仓位/PDT 校验)            │
│   └─ 交易日志模块                                 │
├─────────────────────────────────────────────┤
│  网关抽象层 (vnpy 统一 Gateway 接口)              │  ← 策略代码不感知具体券商/模拟实盘
│   ├─ FutuGateway  (模拟/实盘: trd_env 切换)      │
│   └─ TigerGateway (模拟/实盘: account 号切换)    │
├─────────────────────────────────────────────┤
│  数据层 (历史K线 / 实时行情 / 数据库)              │
└─────────────────────────────────────────────┘
```

**关键设计点**：策略代码只依赖 vnpy 的统一接口，完全不感知"富途还是老虎""模拟还是实盘"。切换只在网关配置层完成。

## 4. 券商对接方案

### 4.1 富途
- 链路：`vnpy_futu → 本地 FutuOpenD 进程 → 富途服务器`
- 部署需常驻 **FutuOpenD**（用富途账号登录）
- 模拟/实盘切换：下单参数 `trd_env = SIMULATE / REAL`（vnpy 网关配置项）
- 模拟盘需先在富途 App 开通"模拟交易"，`get_acc_list()` 才返回 SIMULATE 账户
- 模拟盘特性：免解锁；仅限价/市价单；仅当日有效单；查不到成交明细/费用

### 4.2 老虎
- 链路：`vnpy_tiger → 老虎服务器`（纯 SDK，无需额外进程）
- 凭证：`tiger_id` + RSA 私钥（模拟实盘共用一套）
- 模拟/实盘切换：`account` 账户号格式区分
  - 模拟（Paper）：**17 位数字**，如 `20191106192858300`
  - 实盘环球（Global）：**U 开头**，如 `U12300123`
  - 实盘标准（Standard）：5~10 位数字
- 模拟账号在开发者认证通过后**自动分配**，无需 App 额外开通

### 4.3 凭证准备（由客户方负责）
所有券商账户、API 权限、入金均由**账户本人（客户）**完成，开发方提供申请指引。开发方需要客户提供：
- 富途：可登录 FutuOpenD 的富途账号 + 已开通 OpenAPI + 已开通模拟交易
- 老虎：`tiger_id`、RSA 私钥、模拟账户号（17位）、（上线时）实盘账户号

## 5. 策略一规则代码化（日内 MACD）

> MACD 标准参数（固定）：快线 EMA=12、慢线 EMA=26、信号线 DEA=9

### 5.1 每日开盘前选股（标的池过滤）
全部满足才纳入当日可交易池：
- 日均成交额 ≥ 500 万（港币/美元，按市场）
- 前一日日内振幅：≥2% 且 ≤8%
- 股价 ≥ 2 元
- 排除：当日停牌、即将除权除息、突发重大公告个股

### 5.2 多头开仓（全部满足）
1. 15min：MACD 绿柱持续缩短 + **底背离** + DIF 上穿 DEA 金叉
2. 5min 二次确认：同步 MACD 金叉，无向下空头动能
3. 价格：站上 15min 5 日均线
4. 时间窗：开盘 30 分钟后 ~ 收盘前 90 分钟

### 5.3 空头开仓（全部满足，受可卖空限制）
1. 15min：MACD 红柱持续缩短 + **顶背离** + DIF 下穿 DEA 死叉
2. 5min 二次确认：同步 MACD 死叉，无向上多头动能
3. 价格：跌破 15min 5 日均线
4. 时间窗：同多头
> 注：仅对**可融券做空**的标的生效，需"可空校验"，详见 §8 风险。

### 5.4 出场（优先级：止损 > 止盈 > 尾盘清仓）
- **止损**：多/空单亏损达 1.5% 立即平仓
- **阶梯止盈**：盈利 2% 平 50% 仓；盈利 3.5% 全平；MACD 反向交叉无条件全平
- **尾盘强制清仓**：收盘前 10 分钟全部平仓，零隔夜

### 5.5 仓位与风控
- 单只单次开仓：总资金 10%
- 日内同时持仓 ≤ 3 只，不对同一只加仓
- 当日某标的止损后，禁止再开该标的
- **单日账户最大亏损 3%**，触发后当日停止所有交易
- 信号不纯/无信号不开仓；不重仓、不逆势加仓、不补仓

### 5.6 需在开发期明确数学化定义的项（⚠️ 关键）
以下文档中为"经验描述"，代码实现前必须定为精确规则（建议回测期确定）：
- **背离**：用哪些波峰/波谷、回看多少根 K 线、"价格新低而 MACD 不创新低"的判定阈值
- **绿柱/红柱"持续缩短"**：连续几根判定
- **"站稳/有效破位"5 日线**：是否需收盘价确认、容差幅度

## 6. 策略二规则代码化（中长线选股持仓）

### 6.1 选股条件（全部满足入池）
**月线（定牛熊）**：股价站稳 20 月线、60 月线上方；MACD DIF/DEA 双线在零轴上方
**周线（确认中期）**：5/10/20 周线多头排列；MACD 无顶背离、红柱延续或温和放大；高点低点逐步抬高
**基本面**：近 2 季净利润为正无连续亏损；无退市/立案/重大债务违约；流通市值 港股≥50 亿 / 美股≥20 亿美金
**波动率**：近 3 个月最大回撤 ≤15%

### 6.2 日线择时建仓（入池后）
- 回踩 20 日或 30 日均线企稳
- MACD 无破位死叉，或小回调后再次金叉拐头
- 无放量大跌/利空跳水

### 6.3 出场调仓
- **止损**：月线有效跌破 60 月线且连续 2 月收不回 → 清仓；周线 5 周下穿 10 周多头破位 → 止损；月线 MACD 双线跌破零轴 → 清仓
- **止盈**：盈利 20%~35% 分批减 50%；周/月线 MACD 顶背离 → 清仓；持仓满 6 个月无论盈亏调仓
- **调仓周期**：每月最后一交易日重跑选股公式，剔除失效、纳入新标的

### 6.4 仓位与风控
- 单只最大仓位：总资金 15%
- 总持仓 5~8 只
- 分 2 次建仓：首次 6 成，回踩企稳补满
- 单标的最大回撤 18% 无条件止损
- 短期涨幅 >40% 禁止新建仓
- 趋势未破位忽略短期波动，杜绝频繁交易

## 7. 开发分期计划（建议：回测先行）

| 阶段 | 内容 | 需券商账号 | 产出 |
|------|------|-----------|------|
| **阶段 0** | 环境搭建、vnpy + 网关安装、数据接入（免费历史数据） | 否 | 可运行骨架 |
| **阶段 1（核心）** | 策略一/二规则代码化 + 回测 + 参数优化 + 结果报告 | **否** | 验证两套策略是否盈利的回测报告 |
| **阶段 2** | 富途/老虎**模拟盘**联调：实时行情 + 模拟下单 + 风控 + 日志 | 模拟账号 | 模拟盘全自动运行 |
| **阶段 3** | 前端面板（账户/持仓/信号/日志/参数/开关） | — | 客户可视化产品 |
| **阶段 4** | 实盘小资金试跑 → 验证 → 放量上线 | 实盘账号 | 正式上线 |

**为什么回测先行**：回测不需任何券商账号（只需历史数据，可用 yfinance/akshare 等免费源），且能在写实盘代码、搭前端**之前**就发现规则是否盈利、背离等定义是否合理，避免最大的返工与"上线即亏损"纠纷。

## 8. 关键可行性约束与风险（必须与客户对齐）

1. **美股 PDT 规则**：保证金账户净值 < $25,000 时，5 个交易日内最多 3 笔日内交易。策略一为高频日内，**美股账户须 ≥ $25k 且为保证金账户**，否则美股跑不动。代码内置 PDT 计数与拦截。
2. **做空限制**：港股仅"可沽空名单"内、需借券+报升规则；美股需保证金账户+可借券（locate）。策略一空头**只对部分标的生效**，代码须做"可空校验"，不能假设全市场双向。
3. **背离需数学化**（见 §5.6）：否则"自动识别背离"不可交付。
4. **模拟盘局限**：理想撮合、无真实滑点/流动性、查不到成交费用，**只验证逻辑与链路，不代表实盘盈利能力**，实盘前须小资金最终验证。
5. **数据额度**：5min/15min 历史 K 线深度受富途/老虎额度限制，回测可用历史长度需实测；可用第三方免费源补充。
6. **MT5 不适用**：MT5 基本不支持港美股个股现货，原始需求中"适配 MT5"与富途/老虎路线互斥，本方案采用富途/老虎，不走 MT5。
7. **社区网关版本依赖**：`vnpy_futu`/`vnpy_tiger` 跟随官方 SDK，接入前先用模拟盘验证当前版本可用。

## 9. 前端方案

| 方案 | 成本 | 适用 |
|------|------|------|
| `vnpy_webtrader`（自带 Web） | 最低 | 内部用/对 UI 要求低 |
| **FastAPI + Vue 后台模板 + TradingView 图表（推荐）** | 中 | 交付客户、要像产品 |
| 全自研 React/Vue | 高 | 客户对体验有明确长期要求 |

推荐中间方案：vnpy 引擎跑后台 → FastAPI 暴露 REST/WS → 前端用 vue-vben-admin / Ant Design Pro 套接口 → 行情图用 TradingView Lightweight Charts（免费）。页面含：实时行情、持仓、信号、交易日志、参数配置、策略开关。

## 10. 统一开发规范（来自客户需求）

1. 两套策略完全独立运行、可单独开关
2. 策略一全自动、严格执行日内清仓
3. 策略二全自动选股+提醒，支持半自动建仓、全自动调仓
4. 所有参数（MACD/止盈止损/周期）预留后台修改端口
5. 自带交易日志：每笔开仓/平仓/止损/止盈/选股记录可查
6. 兼容港股、美股全市场数据接口
7. 严格区分：策略一 T0 不隔夜、策略二中长线不高频
